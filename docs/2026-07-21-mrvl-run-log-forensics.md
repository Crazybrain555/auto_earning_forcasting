## 5. Q1 现金流重复计算（头号"机制上不该发生"样本）

- 是什么：DCF 的 FCF 序列把 Q1（截至 5/2）现金流计两次——一次已含在 5/2 资产负债表期初现金（存量），一次又整段进全年前瞻 FCF（流量），存量-流量在 Q1 重叠，抬高每股公允价值。
- 引入点：阶段 F 季度桥改造（L495407）同时喂给资产负债表期初与全年 FCF，单体 build_delivery.py 的 `valuation_metrics()`/`fcf_sequence` 手工接线无"Q1 期间唯一归属"契约（L1636960–1637012）。
- 发现点：只被最后一轮独立红队抓到（L1636973，≈10:27Z）；此前所有确定性校验 PASS（L319798/L1607964）。发现前一行是第 6 次 context compaction。
- 结论：应在契约层杜绝（期间唯一归属 + 存量/流量分离的确定性检查，或写库时唯一约束拦截），而非靠 3300 行手工生成器 + 事后红队兜底。

## 6. 日志膨胀根因

88.8% = exec_output(49.9%) + patch_body(38.9%)。逐项：① 107 次 apply_patch 每次把 build_delivery.py + decision_bundle/method_provenance/run_manifest.json 当"全新文件"整体重写（中位 478KB/~7000 行/次，最大 1.74MB）；② 同套工件 diff 被灌 3 遍（patch 正文 + build stdout echo + collab 回传）；③ 72 次 validate_delivery 各 dump 完整 JSON；④ 反复重读大文件（build_delivery.py 自身 81 次/17.7MB；SEC 原件 64 处）。= 设计(单体+全量重生成) × 引擎(大文件默认全量回显) × 后端(无去重/无截断)。

## 7. 系统设计缺陷清单（驱动 HANDOFF #3）

**SOP/方法**
① 单体 ~3300 行生成器，工件生成未按可独立重建单元拆分→小改触发全量重生成（占 38.9% 日志；违反宪法 #3、#10）。
② 门禁无增量边界：局部缺陷作废整个冻结、重建全套（5–6 轮，占 66% 时间）。
③ 缺 Q1 期间去重契约→存量-流量双计。
④ 检查-构造顺序倒置：先构造带缺陷再靠检查抓（违反宪法 #2）。

**校验器**
⑤ 确定性校验给虚假保证：全 PASS 时 6 个实质构造缺陷仍在——只查恒等式/图/哈希，查不出"公式平衡但经济接线错"。
⑥ 缺三类确定性检查：scenario shock→sheet 实际数据流、FCF 是否 driver-built 而非 back-solved、存量-流量跨期重叠——现在全靠红队人判且太晚。

**引擎（codex）**
⑦ apply_patch 对增长中大文件匹配失败（3 次）→退化整文件重写。
⑧ 大工件默认全量回显（build stdout、collab 回传、校验 dump）。
⑨ collab 并行撞 agent thread limit（L756790）。
⑩ 6 次 context compaction——模型上下文被自产工件挤爆。
⑪ 沙箱小摩擦（rm 被拒、timeout ×2）非主因。

**后端**
⑫ job 日志无体量护栏（132MB 无截断/采样/大 diff 折叠）。
⑬ 部署重启硬杀长跑 job、无检查点/续跑（2h46m 全丢，随即诱发违规重跑 606b）。

**对"瘦身+数据库先行"的支撑**：把单体拆成按工件的小生成单元 + 增量门禁，直接消灭 38.9%（整体重写）与大部分流式 diff/重读；工件落库后以引用/哈希而非全文 diff 流转，关闭四条重复 echo 通道；Q1 双计这类跨期口径重叠可在写库时用唯一约束/去重契约拦截，而非靠红队在被杀前 1 分钟兜底。
